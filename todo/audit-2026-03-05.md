# Murmur Audit Report

**Date**: 2026-03-05
**Auditor**: Codex (codebase review)
**Scope**: `Murmur/*`, `Murmur.xcodeproj/*`, `todo/*`, `ARCHITECTURE.md`, `PRD.md`
**Mode**: Research only (no production code changes)

---

## 1) Executive Summary

Project is close to functional completeness, but **not release-ready** due to several blocking issues in privacy/security and runtime reliability.

### Severity snapshot
- **Critical**: 3
- **High**: 7
- **Medium**: 6
- **Low**: 3

### Key blockers before release
1. Missing `NSMicrophoneUsageDescription` in `Info.plist`.
2. App Sandbox enabled but missing network client entitlement while API backend is implemented.
3. Audio temp files are not consistently removed on failure/error paths.

---

## 2) Findings (Prioritized)

## C-01 — Missing microphone usage declaration (Critical)
**Block**: Privacy safety / Release compliance

**Evidence**
- `Murmur/App/Info.plist` has no `NSMicrophoneUsageDescription` key.
- Mic access is requested in `Murmur/Core/PermissionsManager.swift:28-39`.

**Risk**
- Permission flow may fail at runtime and can trigger process termination behavior on Apple platforms.
- Fails App Review/privacy compliance.

**Recommended fix**
- Add `NSMicrophoneUsageDescription` with clear user-facing rationale.
- Localize the string at least for primary supported locales.

---

## C-02 — Sandbox/API entitlement mismatch (Critical)
**Block**: Security / Functional reliability

**Evidence**
- Sandbox is enabled in `Murmur/App/Murmur.entitlements:5-6`.
- API requests are used in `Murmur/Transcription/OpenAIWhisperService.swift:6-8, 92, 130, 146`.
- `com.apple.security.network.client` entitlement is absent.

**Risk**
- API backend/translation can fail in sandboxed runtime builds.
- Major mismatch with PRD feature set for API translation.

**Recommended fix**
- Add `com.apple.security.network.client = true` for builds that support API mode.
- If local-only distribution is intended, gate API mode out of UI/build.

---

## C-03 — Incomplete cleanup of recorded audio files (Critical)
**Block**: Privacy safety / Error handling

**Evidence**
- Cleanup on success only:
  - `Murmur/Transcription/LocalWhisperService.swift:52`
  - `Murmur/Transcription/OpenAIWhisperService.swift:45`
- Failure path in core flow does not remove file:
  - `Murmur/App/AppDelegate.swift:264-267`
- Runtime recorder error path drops URLs without explicit file removal:
  - `Murmur/Core/AudioRecorder.swift:232-236`

**Risk**
- Voice recordings can remain on disk after failed transcription/paste.
- Privacy leak + disk bloat over time.

**Recommended fix**
- Make one owner responsible for cleanup (`defer` around transcription pipeline).
- Ensure cleanup in every `catch`, cancellation, and runtime error path.

---

## H-01 — Recording state can desynchronize between components (High)
**Block**: Architecture / Thread safety

**Evidence**
- `HotkeyManager` toggles internal `isRecording` eagerly: `Murmur/Core/HotkeyManager.swift:125`.
- `AppDelegate` maintains a separate `isRecordingFlowActive`: `Murmur/App/AppDelegate.swift:26, 214, 236`.
- On start failure, app flag resets but hotkey manager flag is not reconciled: `Murmur/App/AppDelegate.swift:221-230`.

**Risk**
- Extra keypresses, no-op transitions, user-visible inconsistency.

**Recommended fix**
- Keep a single source of truth for recording state in coordinator.
- Hotkey manager should emit events, not own session state.

---

## H-02 — MainActor boundary is implicit in Carbon callback path (High)
**Block**: Thread safety

**Evidence**
- Carbon callback enters `handleHotKeyEvent` and calls `onToggle` directly:
  - `Murmur/Core/HotkeyManager.swift:104-127, 136-140`
- `AppDelegate` is `@MainActor`: `Murmur/App/AppDelegate.swift:8`.

**Risk**
- Potential actor/thread violations and intermittent race behavior.

**Recommended fix**
- Explicitly hop to `MainActor` before invoking UI/coordinator closures from callback.

---

## H-03 — Clipboard-based paste leaks sensitive text to global pasteboard (High)
**Block**: Privacy safety / Security

**Evidence**
- `pasteboard.clearContents()` then `setString(...)` in `Murmur/Core/PasteController.swift:27-30`.

**Risk**
- Transcription text is globally observable by clipboard listeners.
- User clipboard content is destroyed.

**Recommended fix**
- Preserve and optionally restore prior pasteboard contents after paste.
- Add user setting: "Restore clipboard after paste".
- Consider direct insertion path where feasible.

---

## H-04 — Full audio file loaded into memory for API upload (High)
**Block**: Error handling / Performance resilience

**Evidence**
- `Data(contentsOf: audioURL)` in `Murmur/Transcription/OpenAIWhisperService.swift:158`.

**Risk**
- Memory spikes on long recordings, possible OOM/crash.

**Recommended fix**
- Use streamed upload or enforce strict duration/size limits before request.

## M-01 — Stringly-typed configuration keys/values (Medium)
**Block**: Type safety

**Evidence**
- Backend/model/language as raw strings in:
  - `Murmur/Settings/SettingsModel.swift:54-64`
  - `Murmur/App/AppDelegate.swift:67-84`

**Risk**
- Invalid persisted values silently degrade behavior (fallbacks, hidden bugs).

**Recommended fix**
- Replace with enums (`Codable`, `CaseIterable`) + validation on load.

---

## M-02 — Service behavior depends on global singleton state (Medium)
**Block**: Architecture / Testability

**Evidence**
- `OpenAIWhisperService` reads `TranslationConfig.shared.isEnabled` directly: `Murmur/Transcription/OpenAIWhisperService.swift:30`.

**Risk**
- Hidden coupling, harder deterministic testing, temporal bugs.

**Recommended fix**
- Drive behavior from explicit parameters passed by coordinator.

---

## M-03 — API-key status checks are inconsistent (Medium)
**Block**: Error handling / UX correctness

**Evidence**
- UI check uses only non-nil: `Murmur/UI/SettingsView.swift:137-139`.
- Coordinator check trims whitespace: `Murmur/App/AppDelegate.swift:320-327`.

**Risk**
- UI can report "saved" while runtime treats key as missing.

**Recommended fix**
- Centralize validation (`KeychainManager.hasValidAPIKey()`).

---

## M-04 — Logging strategy is print-based and unstructured (Medium)
**Block**: Observability / Readability

**Evidence**
- Multiple `print(...)` across core flow:
  - `Murmur/App/AppDelegate.swift:32,63,76,82,91,206,223,263,359,369,375,379`
  - `Murmur/Core/AudioRecorder.swift:161,201,210`
  - `Murmur/Core/HotkeyManager.swift:132`

**Risk**
- Hard to diagnose field issues, no privacy-safe log policy.

**Recommended fix**
- Migrate to `OSLog` with categories (`audio`, `permissions`, `transcription`, `paste`).

---

## M-05 — Build/toolchain alignment drift (Medium)
**Block**: Architecture / Build readiness

**Evidence**
- Architecture doc states Swift 6 baseline for WhisperKit.
- Project build settings still set `SWIFT_VERSION = 5.0` in `Murmur.xcodeproj/project.pbxproj:235,255,276,297`.

**Risk**
- Team confusion and brittle CI/toolchain behavior.

**Recommended fix**
- Align docs and project settings to one explicit baseline.

---

## M-06 — Warning noise for Intel slice linking (Medium)
**Block**: Build readiness

**Evidence**
- `xcodebuild` emitted linker warnings about arm64 package objects ignored for x86_64 during universal build.

**Risk**
- Intel compatibility uncertain; warning noise hides real issues.

**Recommended fix**
- Decide target architectures explicitly (arm64-only vs universal).
- Ensure dependency outputs match target matrix.

## L-01 — AppDelegate has too many responsibilities (Low)

## 3) Requested Block-by-Block Assessment

## Architecture
- Good: clear component split (`Core`, `Transcription`, `Settings`, `UI`).
- Problems: `AppDelegate` as god object (L-01), state duplication (H-01), singleton coupling (M-02).
- Priority fixes: H-01, M-02.

## Type safety
- Problems: stringly-typed settings/backends/languages (M-01), weakly-typed error payloads (`apiError(String)` blends transport and domain errors).
- Priority fixes: M-01.

## Error handling
- Problems: temp file cleanup gaps (C-03), unstructured logging (M-04), inconsistent API key checks (M-03), memory-heavy upload path (H-04).
- Priority fixes: C-03, H-04.

## Readability
- Problems: overloaded AppDelegate (L-01), print-based logs (M-04), stale docs (L-03).
- Priority fixes: L-01 (incremental refactor plan), M-04.

## Thread safety
- Problems: callback-to-main actor boundary not explicit (H-02), split state machine for recording (H-01).
- Priority fixes: H-02, H-01.

## Privacy safety
- Problems: missing microphone usage string (C-01), leaked temp files on failures (C-03), clipboard exposure (H-03).
- Priority fixes: C-01, C-03, H-03.

## Security (general)
- Problems: sandbox entitlement mismatch for network (C-02), keychain hardening opportunity (L-02).
- Priority fixes: C-02.

## Additional important blocks

### Testing & QA
- No automated tests; manual checks remain open (H-05).
- E1/E2 test tasks not completed (H-06).

### Build/Release readiness
- Toolchain mismatch in docs vs project (M-05).
- x86_64 warning noise in build (M-06).

### Process quality
- Bug log not maintained (H-07).
